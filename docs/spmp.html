1. Introduction

1.1 Overview

GreenThumb is a mobile application that runs on Android devices. It allows users to upload a photograph of a plant and have its species determined by machine learning and browse a database of plants. Further, registered users can upload their photographs to the appropriate species' page, where they can be seen by other users.

This project has no budget.

1.1.1 Objectives

  • Allow users to identify plants with a picture they take with their phone.
  • Allow users to browse a database of plants and plant pictures.
  • Allow users to browse a database of plants and plant pictures.
  • Allow registered users to upload photographs to the speciesâ€™ page, where they can be seen by other users.

  3. Machine Learning Team:

  4. QA Team:
    Stefan Lucic (QA manager), Saad Ansari, Brendan Whelan, Jackson Newlands.

  5. Management Team:
    Nathaniel Carr (project manager), Saad Ansari, Saje Bailey, Tim McGill, Aaron Exley, Stefan Lucic.

The frontend team is responsible for the development of the GUI and integration of GreenThumb onto Android devices.

The backend team is responsible for the development of a database to contain all information pertinent to the application as well as an API to allow communication between the frontend and database.

The machine learning team is responsible for setting up and training the image recognition component of GreenThumb.

The QA sub-team is responsible for assigning test responsibilities to other members of the GreenThumb project and logging the test results.

The management sub-team is responsible for overseeing the progress of the project as a whole.

3.2 Monitoring, Control Mechanisms

The backend, frontend, and machine learning teams have weekly meetings of one hour to discuss scheduling, progress, and issues. The management team has weekly meetings of one hour to review the progress of the project as a whole. The QA sub-team does not have scheduled meetings, but communicates over the messaging platform, Slack.

A shared repository is used to keep all member contributions together as well as document changes to the project and maintain previous versions.

4. Technical Process

  • Frontend: Android Studio, Gimp, Photoshop, Paint.NET

  • Repositories: GitHub
  • Project Management: Github, Google Sheets, Slack, Trello

4.2 Project Staff Training

The managers of each team are responsible for ensuring that each of their team members has the necessary skills to contribute to their part of the project. Individuals with relevant skills on each team are in charge of teaching their teammates, either during the group meetings or during separate meetings which fit the schedules of those involved.

4.3 Quality Control

The QA sub-team designs and assigns tests to the other members of the group and records the results of these tests. In the case that a test fails, the creator of the code is notified by the QA team so changes can be made before the code is re-tested. If the change is minor (eg. a simple syntax error), the tester will fix it themself.

In the case of non-code portions of the product, the QA team assigns an individual who is not the creator to read over the writing and compare it with the official style guide as well as check for any spelling or grammar mistakes. This person must fix any inconsistencies and log what changes were made to improve the document.
