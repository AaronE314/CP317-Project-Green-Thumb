1. Introduction

The design document provides documentation that will assist in the software development of GreenThumb by providing blueprints for how the software must be built. This document contains standards such as the planned implementation, interface guidelines, detailed design, and maintenance. The design document serves as a reference for the overall design of Greenthumb.

1.1 Purpose

The purpose of this document is to establish design standards that the programmers and designers must adhere to. Consistency in the structure and design of this application will help to avoid future collaboration trouble and alleviate maintenance struggles.
