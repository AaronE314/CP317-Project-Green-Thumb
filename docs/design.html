1. Introduction

1.1 Overview

GreenThumb is an Android application that allows users to upload photographs of plants and have their species identified. Users logged in with their Google account are able to upload their photographs to the pages of the corresponding plant and vote on the photographs of other signed-in users. Additionally, a user may search a plant by its name.

1.2 Purpose

The purpose of this document is to establish the architectural and detailed design for the developers to follow.

1.3 Intended Audience

The intended audience of this document is the developers.

2. Implementation Plan

2.1 Choice of Language

On the Android side of the app, Java is used, as it is the primary language used for Android development. Consequently, there is a great deal of well-designed documentation available from which to learn.

On the backend side of the app, the Node.js library for JS is used for the server, as it is one of the industry standards for servers. There is a great deal of documentation on API development for Node.js, and setting up server communication is relatively easy.

For the machine learning portion of the app, the Tensorflow library is used. Tensorflow supports Python, and JavaScript, but the primary library from Tensorflow here implemented only works on the Python version of Tensorflow. Conequently, the retraining of the network is done through the Python version, while the rest of the machine learning is done via JavaScript using Tensorflow.js.

For the Database, MS SQL Server is used because of its excellent RDBMS functionality and support for the programming languages used by the other components of the application. Another advantage of SQL Server over similar products like MySQL is the ability to stop query execution midway without killing the entire process.

2.2 Integration

The user login and logout functions are provided through integration with the Google Sign-In API. Both the Android and web frontend communicate with the backend via a RESTful API created using Express.js. This API then operates on the database directly and interacts with the machine learning model through the MLTrainer and MLIdentifier. The MLTrainer and MLIdentifier use the Tensorflow object detection library.
