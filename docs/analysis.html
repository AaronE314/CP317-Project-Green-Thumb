1. Introduction

Green Thumb is a mobile application built for Android devices. It allows users to upload a photograph of a plant and have its species determined by image recognition and machine learning, as well as browse a database of plants to learn more about them. Furthermore, registered users can upload their photographs to the appropriate speciesâ€™ page, where they can be seen by other users.

2. Object Classification

2.1 Entity Objects

  • Admin - A special type of User which has certain privledges. They can ban other Users; add, update, and remove Plants; and handle PhotoReports.
  • Ban - A restriction on users placed by Admins.
  • Photo - A photograph of a Plant subject to upvotes and downvotes.
  • PhotoReport - A User-submitted report of a Photo to be handled by an Admin.
  • Plant - The details for one species of plant including a name and biography.
  • User - A user of GreenThumb that has signed in with their Google account. They can post Photos and vote on the Photos posted by other Users.

2.2 Boundary Objects

  • LoginActivity - A welcome screen shown to a user who has never opened GreenThumb before. Provides a choice between logging in with the user's Google account or continuing as a guest.
  • GalleryActivity - A screen containing multiple photos of a species of plant.
  • MainMenuActivity - The primary screen from which the user can navigate to the other screens.
  • PlantBioActivity - A screen containing information on the selected plant.
  • ReportsActivity - A screen for reporting a photo.
  • ScanActivity - A screen wherein the user can take and send to the server photos of their plants for identification.
  • SearchActivity - A search screen to browse plants by name.
  • SettingsActivity - A settings screen wherein a guest can log in if they change their mind and change other settings as options are added.
  • SplashScreenActivity - A welcome screen shown to a user who has opened GreenThumb before. A splash screen displayed while the HomeView is loading.

2.3 Control Objects

  • AppServer - Responds to HTTP requests sent by the views to the backend.
  • MLIdentifier - Identifies the plant in a user-submitted photo.
  • MLTrainer - Trains the image recognition model by feeding it user-submitted, admin approved, photos and an Record file corresponding to the chosen images.
