1. Introduction

Green Thumb is a mobile application built for Android devices. It allows users to upload a photograph of a plant and have its species determined by image recognition and machine learning, as well as browse a database of plants to learn more about them. Further, registered users can upload their photographs to the appropriate speciesâ€™ page, where they can be seen by other users.

2. Object Classification

2.1 Entity Objects

  • Admin - A special type of User that is able to strike/ban other Users, add/update/remove Plants, and handle PhotoReports.
  • Photo - A photograph of a Plant subject to upvotes/downvotes by Users.
  • PhotoReport - A User-submitted report of a Photo to be handled by an Admin.
  • Plant - The details for one species of plant, including a name, bio, and region, as well as a set of photographs of the species.
  • User - A user of GreenThumb that has signed in with their Google account. They can post Photos to Plants, and upvote/downvote the Photos of other Users.

2.2 Boundary Objects

  • AdminView - A web page wherein Admins can perform administrative actions.
  • FirstStartUpView - A welcome screen shown to a user who has never opened GreenThumb before. Provides a choice between logging in with the user's Google account or continuing as a guest.
  • GalleryView - A screen containing multiple photos of a species of plant.
  • HomeView - The primary screen from which the user can navigate to the other screens.
  • PhotoReportView - A screen for reporting a photo.
  • PlantView - A screen containing information on the selected plant.
  • ScanView - A screen wherein the user can take and send to the server photos of their plants for identification.
  • SearchView - A search screen to browse plants by name.
  • SettingView - A settings screen wherein a guest can log in if they change their mind.
  • StartUpView - A welcome screen shown to a user who has opened GreenThumb before. A splash screen displayed while the HomeView is loading.

2.3 Control Objects

  • AppServer - Responds to HTTP requests sent by the views to the backend.
  • MLIdentifier - Identifies the plant in a user-submitted photo.
  • MLTrainer - Trains the image recognition model by feeding it user-submitted photos.
